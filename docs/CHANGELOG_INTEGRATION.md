# Журнал изменений интеграции (adminka)

## POST `/api/report-templates`

- **Опционально:** в теле JSON можно передать `briefList: true` (или `brief_list: true` — нормализуется как и прочие поля на стороне клиента с camelCase).
- **Поведение:** ответ сохраняет ту же структуру элементов (`reportTemplateId`, основные поля шаблона и т.д.), но **не** возвращает тяжёлые данные: `sqlQuery` — пустая строка, `reportInfo` — пустой объект, `organizations` / `accessGroups` / `recipients` — пустые массивы. Полная детализация по-прежнему доступна через существующие GET по шаблону.
- **Совместимость:** без флага поведение без изменений.
