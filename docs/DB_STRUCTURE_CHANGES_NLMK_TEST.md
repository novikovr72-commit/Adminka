# Изменения структуры БД `nlmk_test` (проект `adminka`)

## Назначение

Этот файл — единый журнал **всех** изменений структуры БД `nlmk_test`, которые выполняются в рамках проекта `adminka`.

Обязательное правило:
- при любом изменении структуры (таблица/колонка/индекс/constraint/comment/view/function/trigger) добавлять новую запись в этот файл в день изменения;
- для таблиц с полем `name_eng` вносить данные так, чтобы `name_eng = name` для той же записи.

## Формат новой записи (обязательно)

Для каждой новой структурной правки добавлять блок:

```md
### YYYY-MM-DD — <краткое название изменения>
- Контур: <schema.table / объект>
- Тип: <CREATE/ALTER/DROP/INDEX/COMMENT/...>
- SQL-файл: <путь к sql-файлу или "inline migration">
- Изменение:
  - ...
- Обратная совместимость:
  - ...
- Влияние на backend/frontend:
  - ...
- Проверка:
  - ...
```

## История изменений

### 2026-03-11 — создание хранилища шаблонов печатных форм
- Контур: `public.print_form_templates`
- Тип: `CREATE TABLE` + индексы
- SQL-файл: `adminka/add-print-form-templates.sql`
- Изменение:
  - создана таблица `public.print_form_templates`;
  - добавлены поля: `id`, `code`, `name`, `name_eng`, `description`, `template_pdf`, `data_sql`, `field_mapping`, `overlay_settings`, `deleted`, `created_at`, `updated_at`;
  - добавлен уникальный индекс `ux_print_form_templates_code` (для `code`, только для `deleted = false`);
  - добавлен индекс `ix_print_form_templates_deleted`.
- Обратная совместимость:
  - изменение не ломает существующие таблицы/методы.
- Влияние на backend/frontend:
  - поддержка отдельного раздела печатных форм.
- Проверка:
  - таблица и индексы существуют;
  - запись шаблона создается/читается/обновляется.

### 2026-03-12 — добавление поля DaData в карточку организации
- Контур: `party.organ_unit`
- Тип: `ALTER TABLE`
- SQL-файл: `adminka/add-organ-unit-data-info.sql`
- Изменение:
  - добавлена колонка `data_info jsonb`.
- Обратная совместимость:
  - `IF NOT EXISTS`, безопасно для повторного запуска.
- Влияние на backend/frontend:
  - хранение и отображение ответа DaData в карточке организации.
- Проверка:
  - `data_info` доступно в `GET /api/admin/organization/{organUnitId}`;
  - `POST /api/admin/organization/{organUnitId}/dadata/refresh` сохраняет JSON в `data_info`.

### 2026-03-13 — добавление логотипа шаблона отчетов
- Контур: `public.report_templates`
- Тип: `ALTER TABLE` + `COMMENT`
- SQL-файл: `adminka/add-report-logo-to-report-templates.sql`
- Изменение:
  - добавлена колонка `report_logo bytea`;
  - добавлен комментарий к колонке.
- Обратная совместимость:
  - `IF NOT EXISTS`, безопасно для повторного запуска.
- Влияние на backend/frontend:
  - возможность хранить логотип и использовать его в выгрузке отчетов.
- Проверка:
  - чтение/запись `report_logo` в карточке шаблона отчета работает.
