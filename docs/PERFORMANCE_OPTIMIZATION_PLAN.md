# План оптимизации отклика UI (adminka)

## Контекст

Основной рендер сосредоточен в монолитном `App.jsx`; любой `setState` в корне трогает всё дерево. Списки (в т.ч. «Настройка отчётов») страдают от того же.

## Как проверять эффект

Chrome **Performance** + **React Profiler** до/после Фазы 1.1: число коммитов `App` за 10 с простоя должно стать **0** (раньше ~10/с из‑за часов).

**Профилирование после фаз 2–5:** сценарии «открыть список шаблонов → открыть карточку → правка SQL / вкладка шаблон» — в Profiler сравнить число коммитов корня и время рендера; после Фазы 6 ожидается снижение коммитов `App` при работе только с отчётами.
